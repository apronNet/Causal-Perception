Why These Controls Exist

Direct launching is strongest when the moving ball reaches the target and the target begins moving with little or no delay. Add a spatial gap, full overlap, or a long pause, and the event starts to lose its causal look.

Preset Notes

The presets recreate a canonical launch, a delayed launch, a gapped event, a causal-capture display, and an occluded tunnel event with hidden launching.

How To Use

A fast workflow for stimulus design
1

Start From A Preset

Choose the closest literature family, then click Apply Preset. Use canonical launching as the baseline, overlap presets for launch/pass continua, and occluded presets for tunnel cases.

2

Tune The Event Structure

Adjust speed, acceleration, delay, gap or overlap, and launcher behavior first. These controls change the causal display itself, so treat them as experimental variables.

3

Add Context Carefully

Use context events, tunnel occlusion, depth order, and sound only when they are part of the hypothesis. The standards cards above show overlap, timing, category, and capture status.

4

Export Clean Stimuli

Switch to clean stimulus or fixation mode before exporting participant-facing clips. Export the WebM and the JSON sidecar so every parameter can be reported or preregistered.

Next Research Directions

What the current literature suggests testing next

Map The Boundary

Sweep overlap and delay together to estimate where reports shift from launch to pass or delayed launch. This is the safest next psychometric baseline.

Test Locality

Use event offsets to compare same-location and shifted-location clips. This supports retinotopic adaptation designs rather than treating adaptation as a generic response bias.

Separate Features

Cross direction, speed, and color identity. Recent adaptation work suggests direction is more diagnostic than speed or color for launch routines.

Pull Apart Categories

Compare launching, triggering, entraining, and single-object controls. This asks whether causal perception is one category or several related event kinds.

Context After Adaptation

Combine ambiguous test events with launch, pass, and no-context trials before and after adaptation. This targets whether contextual capture bypasses local launch detectors.
